# 半自動消失線身高估測工具之實作與分析：基於 Java 的圖形化介面設計

Implementation and Analysis of Semi-Automatic Vanishing Line Height Estimator: An Object-Oriented Design Based on Java

## 專題摘要

本專題以 Java 實作一個基於圖形化介面的半自動身高估測工具，利用單張照片中的消失線與人物像素位置建立比例關係，並以已知身高的基準人物推估同張照片中其他人物的身高。系統採半自動點選方式蒐集關鍵座標，可降低人工輸入成本，同時保留演算法流程的可解釋性。

## 研究目標

1. 以單張照片中的透視資訊建立相對比例模型。
2. 以已知身高基準人物（180 cm）推估其他人物身高。
3. 透過滑鼠點選完成座標蒐集，減少手動輸入負擔。
4. 分析主要流程的時間與空間複雜度，評估方法可行性。

## 系統設計

本專案以物件導向方式組織程式結構，核心元件如下：

1. `PixelPoint`
- 代表像素座標點 `(x, y)`。

2. `Person`
- 代表人物資料，包含 `name`、`head`（頭頂點）、`foot`（腳底點）。

3. `ClickPurpose`
- 定義點選順序，包含：
	- `HORIZON_A`、`HORIZON_B`
	- `REF_HEAD`、`REF_FOOT`
	- `TARGET_HEAD`、`TARGET_FOOT`

4. `ClickCollector` 與 `ImagePanel`
- `ClickCollector`：負責建立視窗、收集滑鼠點選，並同步等待使用者完成操作。
- `ImagePanel`：負責顯示圖片、標示已點選點位，以及繪製消失線參考線。

## 操作流程

1. 啟動程式並輸入照片數量。
2. 針對每張照片輸入圖片路徑、目標人物數量與目標姓名。
3. 開啟圖片視窗，依序點選：
	 - 消失線點 A
	 - 消失線點 B
	 - 基準人物頭頂與腳底
	 - 各目標人物頭頂與腳底
4. 計算人物投影比例：
	 - `pixelHeight = |y_foot - y_head|`
	 - `horizonY(x_foot)`
	 - `ratio = pixelHeight / |y_foot - horizonY(x_foot)|`
5. 以基準人物換算目標人物身高：
	 - `H_target = 180 * (ratio_target / ratio_ref)`
6. 輸出每位目標人物的估測結果（cm）。

## 時間複雜度分析

令：

- `n` = 單張照片中目標人物數量
- `k` = 單張照片總點選數，且 `k = 4 + 2n`

主要步驟的複雜度如下：

1. 點選資料收集
- 每次點選僅記錄一次座標，時間複雜度為 `O(k)`。

2. 每位人物估測
- `projectedHeight()` 與 `yOnLine()` 皆為常數時間，單一人物為 `O(1)`。
- `n` 位目標人物總計為 `O(n)`。

3. 單張照片整體
- 讀圖與繪圖屬影像處理成本；就估測主流程而言，整體為 `O(k + n)`。
- 由於 `k = 4 + 2n`，可視為線性時間 `O(n)`。

空間複雜度方面，儲存點位與人物資料為 `O(k + n)`，整體亦可視為 `O(n)`。

## 準確度建議

1. 頭頂點應選擇最高可見點，腳底點應選擇接地點。
2. 消失線 A/B 建議拉開距離，避免線段過短造成不穩定。
3. 同張照片中的所有人物應使用同一條消失線。
4. 若腳點接近消失線，分母接近 0，誤差會明顯放大。

## 結論

本實作以半自動點選方式完成單張照片多人身高估測，兼顧操作性與可解釋性。透過基準人物（180 cm）與透視比例轉換，可在不需相機內參的情況下進行相對身高推估。主流程在每張照片上為線性時間，適合課堂作業與中小型資料量的實驗分析。
